# Module 3 – Control Flow & Logic (The CPU's Perspective)

Decision-making, branching, and loops – how the CPU changes the instruction pointer.

## Projects

- `project06-temp_convertor` – Fahrenheit ↔ Celsius with strict data types
- `project07-weight_convertor` – Pounds ↔ Kilograms using `#define` macros
- `project08-voting_eligibility_checker` – Direct comparison output (no `if`)
- `project09-number_duel` – Player vs CPU using hardcoded logic (later upgraded with random)
- `project11-calculator` – Switch‑based arithmetic with operator input
- `project13-countdown_timer` – OS‑level sleep call (`<unistd.h>`) for timed loops
- `project14-multiplication_grid_table` – Nested loops and tab formatting
- `project15-survival_loop_game` – State machine with `while`, `switch`, `break`, `continue`
- `project16-atm_machine` – Auth loop → event loop → dispatcher (shell/game engine pattern)
- `project17-packet_loss_simulator` – Fault tolerance with retry logic (TCP‑style)
- `project18-dungeon_explorer` – Capstone RPG: random events, combat, state management

## Covered

- `if`/`else if`/`else` and block scope
- Comparison operators, truth in C (0 vs non‑zero)
- Logical operators (`&&`, `||`, `!`) and short‑circuit evaluation
- `switch`/`case` with `break` and fall‑through
- Ternary conditional `? :`
- `while`, `do-while`, `for` loops
- `break` and `continue`
- Nested loops and O(n²) patterns
- Random numbers: `rand()`, `srand(time(NULL))`, range with modulo
- `goto` for error handling (Linux kernel style)
- Input validation and buffer clearing
